# Autolycus Agent v0.1.2 Release Notes

## What's New

### Interactive Prompt Detection (feat: terminal)
When a foreground terminal command outputs an interactive prompt, the process is automatically paused and registered with `process_registry` so the agent can respond via `process(action='submit')`. This enables handling of interactive CLI tools like `gh auth login`, `sudo`, package installers, etc.

**Prompt patterns detected (9 total):**
- Password prompts (`Password:`, `passwd:`)
- Yes/no confirmations
- Continue/proceed prompts with [y/n]
- "Are you sure?" questions
- Preference prompts ("Do you want...", "Would you like...")
- REPL prompts (`> `, `>>> `, `. `)
- Input requests ("Enter <something>", "Type <something>")

### Changes
- `tools/environments/base.py`: added `_PROMPT_PATTERNS` regex and detection hook in `_wait_for_process()` poll loop
- `tools/process_registry.py`: added `register_paused_process()` method to wrap existing Popen handles with a reader thread
- `tools/terminal_tool.py`: integrated paused process registration, returns JSON hint to agent when prompt detected
- `tests/tools/test_prompt_detection.py`: pytest-based unit tests for all patterns
- `tests/tools/test_prompt_detection_manual.py`: standalone integrity check (no external deps)

### Branch Note
This release is on the `lycus` branch — diverged from upstream Hermes to maintain Lycus-specific features independently.
