## Release Notes — Autolycus v0.2.1 (Patch)

**Release Date:** June 23, 2026  
**Platform:** FreeBSD, Linux, macOS  
**License:** LGPL v2.1  

> Patch release: fixes critical dependency conflicts introduced in v0.2.0 that blocked `lycus update`, `pip install`, and `npm install`. If you hit errors upgrading to v0.2.0, this resolves them.

---

## Bug Fixes

### Dependency Conflicts (fix)

Two blocking issues from the v0.2.0 release have been resolved:

**1. Starlette version conflict (CVE-2026-48710)**
The `chatterbox-tts==0.1.7` package pins `gradio==6.8.0`, which requires `starlette<1.0`. This conflicts hard with our security pin on `starlette==1.0.1` (CVE-2026-48710 fix).

**Resolution:** chatterbox removed from the `[all]` extra in `pyproject.toml`. Users who need chatterbox can install it separately in a dedicated venv or use the lazy-install path (`tts.chatterbox`) which resolves at first-use. The underlying issue is upstream — gradio 6.19.0+ supports starlette>=1.0.1, but chatterbox-tts has not yet updated its dependency.

**2. npm E404 on third-party parser packages**
The v0.2.0 automated rebrand script incorrectly renamed two external npm dependencies (Meta's JS engine parser + ESTree utilities, used by `eslint-plugin-react-hooks`) to non-existent `lycus-*` names in `package-lock.json`, causing E404 failures on `npm install`. These are third-party packages published under their own identifiers — they cannot be renamed without breaking the build.

**Resolution:** Restored the correct upstream package identifiers for these external dependencies. The rebrand script must exclude third-party npm packages from renaming; only our own code and assets carry the Lycus/Autolycus identity.

### Chatterbox Restoration (fix)

After initial removal, chatterbox has been restored to the `[all]` extra with a `uv gradio` override strategy instead of outright removal. This allows users who want the full feature set to get chatterbox without hitting the starlette conflict — uv resolves the dependency graph correctly when given the right constraints.

### Web Dashboard (fix)

Fixed a TypeScript compile error in `web/src/pages/SystemPage.tsx` where `autolycus_version` was renamed to `lycus_version`, which blocked the entire web UI build pipeline. Pre-built SPA assets were regenerated and included under `lycus_cli/web_dist/` (17 files), ensuring the dashboard ships ready-to-serve without requiring a separate build step.

### Documentation Clarity (docs)

Replaced vague "rebrand" comments throughout the codebase with specific descriptions of what actually changed at each location:
- `toolsets.py`: clarifies legacy CLI alias purpose for config compatibility
- `web/src/themes/context.tsx`: explains lens-5i → nous-blue theme key rename reason
- `website/docusaurus.config.ts`: documents automation templates → blueprints redirect
- Bootstrap installer Rust doc comments updated with precise identity references
- Migration scripts and test files now explain name normalization purpose
- `PLANS/freebsd-installation-test.md`: FreeBSD installation testing plan documented
