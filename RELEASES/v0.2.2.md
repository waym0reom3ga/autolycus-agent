## Release Notes — Autolycus v0.2.2 (Feature)

**Release Date:** July 23, 2026  
**Platform:** FreeBSD, Linux, macOS, Armbian (armv7l)  
**License:** LGPL v2.1  

> Feature release: TotalRecall memory system replaces OpenViking as the default memory provider. Khronos workflow orchestration integrated into installer. Full armv7l/32-bit ARM support for Raspberry Pi and single-board computers. Upstream compressor, security, and performance improvements cherry-picked from Hermes.

---

## New Features

### TotalRecall Memory System (feat)

TotalRecall is now the default memory provider, replacing OpenViking with a more robust architecture:

- **Semantic tag extraction** — commits automatically extract meaningful tags from content for faster recall
- **FTS5 fallback recall** — full-text search backup when semantic matching fails
- **Lifecycle hooks** — `on_session_switch` and `on_pre_compress` hooks allow plugins to react to memory state changes
- **Compression error handling** — graceful degradation with failure counting instead of hard crashes
- **Migration tool** — automatic migration from OpenViking/legacy memory formats

### Khronos Workflow Orchestration (feat)

Khronos workflow server is now integrated into the installer:

- **Temporal compatibility** — durable job scheduling with Temporal workflow engine on port 7233
- **Circuit breaker and retry limits** — resilient connection handling for long-running workflows
- **Thin wrapper API** — minimal overhead bridge between Khronos and existing cron infrastructure

### armv7l / 32-bit ARM Support (feat)

Full installer support for Raspberry Pi, Armbian, and other 32-bit ARM platforms:

- **System Python detection** — installer now detects and uses system Python before falling back to bundled version
- **Python 3.13 default** — upgraded from 3.11 with proper venv creation on armv7l
- **Build dependencies** — automatic installation of `libjpeg-dev`, `zlib1g-dev` for Pillow compilation
- **Lockfile sync skip** — avoids incompatible lockfile operations on 32-bit architectures

### USB Andon Tower Light Signaling (feat)

Hardware integration for USB tower lights:

- **Lifecycle signaling** — visual status indicators tied to agent state changes
- **Independent subprocess timer** — 30-second auto-off with dedicated process isolation
- **Dynamic device discovery** — automatic detection of connected tower light hardware

### Persistent Video Streaming (feat)

POSIX computer use now supports persistent video streams:

- **Dynamic device discovery** — automatically finds available capture devices
- **Continuous streaming** — maintains connection without requiring re-initialization

### Identity Splitting: SOUL.md + MASK.md (feat)

Agent identity is now split into two layers:

- **SOUL.md** — immutable core identity that persists across sessions
- **MASK.md** — role layer that can be swapped for different personas

## Bug Fixes

### TotalRecall Compression Loop Resolution (fix)

Multiple fixes to resolve compression loop issues:

1. **Anti-thrashing bypass** — prevents LLM budget exhaustion from repeated compression attempts
2. **Session end caps** — hard limits on compression cycles per session with `fail_count` tracking
3. **Permanent skip mechanism** — content that consistently fails compression is permanently skipped
4. **Prefetch cap and compressible guard** — early detection of non-compressible content before LLM calls

### Installer Fixes (fix)

- Restored gateway_ws and dashboard functions accidentally truncated during cron consolidation
- Correct system Python path resolution for venv creation on armv7l
- Proper error logging with pip stderr capture for debugging installation failures

### Upstream Cherry-Picks from Hermes (fix/security/perf)

Cherry-picked critical fixes from upstream NousResearch/hermes-agent:

**Compressor fixes:**
- Treat empty-content summary response as failure (#50297)
- Pin summary role to user when only system prompt is protected (#52160)
- Skip context-summary markers as last-user tail anchor
- Strip orphan tool_calls instead of inserting stubs (#51218)
- Skip compression during summary LLM cooldown to prevent CLI freeze
- Prevent orphan user turn after compaction via turn-pair preservation

**Security patches:**
- Scope Matrix /resume by thread, not just room (IDOR fix)
- Scope /resume and /sessions to caller's origin (IDOR fix)
- Strip VERTEX_CREDENTIALS_PATH/GOOGLE_APPLICATION_CREDENTIALS from subprocess env
- Narrow credential-query denylist to unambiguous names

**Performance improvements:**
- Lazy-load gateway platform adapters (#54448) — faster startup
- Parse config + plugin manifests with libyaml CSafeLoader (#54486) — reduced memory usage
- Write config.yaml as UTF-8 to prevent emoji/personality corruption (#51676)

### OpenViking Ports (feat/fix)

Ported from upstream before TotalRecall replacement:
- Trusted identity retry mechanism
- `viking_forget` tool for selective memory removal
- Memory mirror worker drain on teardown
- All memory-provider API keys catalogued in OPTIONAL_ENV_VARS

## Refactoring

### Memory Condenser Unification (refactor)

The standalone `memory_condenser` has been unified with TotalRecall:

- **Reduced from 732 to ~100 lines** — eliminated redundant compression logic
- **Migration tool included** — automatic conversion of existing condensed memory
- **Hardcoded values removed** — IP addresses, API keys, model names, and context sizes now configurable
- **LLM thinking preamble stripped** — cleaner condensation output without reasoning artifacts
- **500 token limit removed** — allows full condensation output when needed

### Cron Consolidation (refactor)

Jobs and scheduler consolidated into `temporal_bridge` with thin wrapper API:

- Single source of truth for job scheduling
- Reduced code duplication between cron subsystems
- Maintained backward compatibility with existing job definitions

### Code Cleanup (refactor/chore)

- Deep cleanup of rogue agent rebrand remnants
- Removed standalone Hermes launcher that conflicts with Meta's Hermes bytecode compiler
- Restored TotalRecall plugin directory after accidental removal
- Corrected misleading tips in lycus_cli
- Cleaned up stale .gitignore entries
- Removed accidentally committed 3MB Dovecot .deb package

## Upgrade Notes

- **TotalRecall is now the default memory provider.** If you were using OpenViking, run the migration tool: `lycus migrate-memory --from openviking`
- **armv7l users:** The installer will automatically detect your system Python and install required build dependencies. No manual configuration needed.
- **Khronos integration:** Requires port 7233 available. If you have another service on this port, update the Khronos config before running the installer.
