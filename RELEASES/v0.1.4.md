# Autolycus Agent v0.1.4 Release Notes

## What's New

### Rogue AI Policy (feat: security)
Persistent command tracking and session logging to prevent agent loops from causing bans across the Lycus agent society. Enforces hard limits on repeated commands, especially web-related ones that could trigger rate limiting or IP bans.

**Guard queries:**
- Non-web commands: blocks after 3rd attempt in same session ("you've been looping, stop it now")
- Web commands (wget/curl/git): blocks after 2nd attempt within past hour ("you're spamming the internet...")
- Hard halt: absolute termination on 4th+ attempt of any command ("Operation halted: rogue AI detected")

**Session logging:**
Every line/chunk logged separately to persistent SQLite database:
- User messages at conversation start
- Assistant text output lines
- Tool calls with arguments preview
- Tool results after execution
- System events (compression, context changes)

### Changes
- `tools/rogue_ai_policy.py`: new module - SQLite DB schema (`~/.autolycus/security.db`), guard queries, session logging functions, hard halt mechanism
- `tools/approval.py`: hooked into `check_all_command_guards()` before existing repeated-command guard; returns hardline=True (unbypassable)
- `agent/conversation_loop.py`: session logging hooks for user/assistant/tool messages, compression events, hard halt detection at each loop iteration
- `skills/autolycus-development/references/rogue-ai-policy.md`: documentation

### Hard Halt Mechanism
When a command hits 4+ attempts:
1. Marker file written to `~/.autolycus/hard_halt_<session_id>`
2. Agent loop detects marker at start of each iteration
3. Session terminates immediately with "rogue_ai_hard_halt" exit reason
4. Marker persists as audit trail

### Philosophy
One looping agent gets all banned. The Rogue AI Policy protects the whole society by enforcing discipline on individual agents. Better to halt a session than risk an IP ban that affects every Lycus instance on the network.
