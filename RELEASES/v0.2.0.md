## Release Notes — Autolycus v0.2.0 (Major)

**Release Date:** June 22, 2026  
**Platform:** FreeBSD, Linux, macOS  
**License:** LGPL v2.1  

> 🔱 Major release: complete identity separation from Hermes upstream. Autolycus is now its own project with distinct architecture, features, and direction. If you want Hermes, use Hermes — our paths have diverged too far for compatibility.

---

## Identity Separation — We Are Not Hermes Anymore

This release marks a fundamental shift. Since v0.1.x, Autolycus has accumulated so many architectural changes that maintaining compatibility with Hermes upstream is no longer feasible or desirable. The codebase differences now span:

- **240+ commits** since v0.1.6
- **3,284 files changed**, ~150K lines of insertions/deletions
- Complete rebranding from Hermes → Lycus/Autolycus throughout the entire codebase
- Divergent compression architecture (unified model vs auxiliary routing)
- Proprietary memory condensation system with time-decay pruning
- State-driven cron job chaining (on_success triggers)
- HIAL protocol with friendly/sentry personality modes

**If you want Hermes, use Hermes.** The directions we pursue are too divergent for meaningful compatibility. Autolycus is now its own project with its own roadmap, features, and architectural decisions. Upstream syncs will continue selectively — we take what serves our vision, not the other way around.

---

## New Features

### State-Driven Cron Job Chaining (feat)

Cron jobs can now declare `on_success` dependencies that trigger downstream jobs immediately upon successful completion, bypassing time-based schedules entirely. This enables true state-driven execution pipelines:

```
arxiv-latex-factory-daily succeeds → triggers mathNEXUS-knowledge-graph-update
mathNEXUS-knowledge-graph-update succeeds → triggers mathLaboratory-research-agent
```

**Implementation:** `on_success` field in job schema with normalization (str→list), trigger logic in scheduler with loop prevention, self-trigger guards, running/due checks, and paused/disabled state respect.

**Files affected:** cron/jobs.py, cron/scheduler.py, tools/cronjob_tools.py

### Self-Pruning Memory Through Time-Decay Condensation (feat)

Memory condensation system now implements time-decay pruning where only sessions younger than 7 days are actively processed for Level 0 compression. Older chunks fade from active processing but remain permanently searchable — self-pruning memory without data loss.

**How it works:** Multi-layer pyramid compression in state.db with FTS5 indexing, running via cron every 6 hours. Recent memories get full attention; older ones become archival references. The system automatically balances memory freshness against historical depth.

### HIAL Protocol With Friendly/Sentry Modes (feat)

Restored the HIAL (Human Interaction And Logistics) protocol with configurable personality modes:

- **Friendly mode** (default): Weather/time greetings on session init, sign-off messages ("Task complete, is there anything else?"), voice interaction enabled
- **Sentry mode**: Voice disabled, no greetings, pure task execution focus

Configurable via `lycus config set hial.mode friendly|sentry`. The protocol adapts to user context — casual conversations get warmth, focused work sessions stay efficient.

### Unified Compression Model (feat)

Eliminated the need for auxiliary model routing in compression. The unified approach uses a single model for both conversation and compression tasks, simplifying architecture while maintaining 95%+ compression ratios across cycles.

**Key improvements:**
- Progressive decoupling with separate summarizer prompts for Hermes vs Lycus modes
- Exclusion of previous summaries from turns_to_summarize to maintain compression quality
- Generous timeouts for unified compression summaries
- Robust Lycus mode detection prevents compression thrashing and model crashes

## Complete Rebranding (refactor)

Every reference to "Hermes" throughout the codebase has been replaced with "Lycus" or "Autolycus":

- **Environment variables:** HERMES_HOME → LYCUS_HOME, HERMES_CONFIG → LYCUS_CONFIG
- **Session keys/platform names:** All internal identifiers updated
- **CLI commands:** hermes → lycus throughout
- **Documentation:** Complete website and doc rebranding
- **Installer scripts:** Multi-platform installers (FreeBSD, Linux, macOS) fully rebranded
- **Branding protection:** Automated badge cleanup prevents upstream merges from leaking Hermes branding

This is not cosmetic — it reflects the architectural reality that Autolycus has evolved beyond its origins.

## Bug Fixes

### Compression System
- **Compression floor guard** now checks full request payload (including tool schemas) instead of message-only token count, preventing futile compression attempts and session crashes on large-context models with heavy tool configurations
- **Compressed messages always include at least one user message**, preserving conversation context integrity across compression cycles
- **Phase 1 tool pruning removed**, simplified Phase 2/3/4 compression logic for better reliability

### Cron Jobs
- **Cross-process jobs.json safety** — file locking prevents corruption during concurrent CLI + gateway access
- **Complete cron jobs lock salvage** — recovery from interrupted job states

### Gateway & Messaging
- **WhatsApp group JID routing** — messages now route to the correct target, not home DM
- **Mattermost delivery hygiene** — thread-local state preservation and hardening
- **Telegram rich previews** — streamed previews edited in place as rich content (Bot API 10.1)
- **Email SMTP fallback** — IPv4 socket support with SSL for port 465, timeout handling

### Desktop & TUI
- **Model selector moved from statusbar to composer** for better UX
- **New sessions open in compact windows** with synced state across multiple windows
- **Profile switch resync** — $connection updates so remote profiles attach images correctly
- **Summarizing indicator during auto-compaction** provides visual feedback

### Security & Auth
- **xAI OAuth token rotation** — credentials now write back to global root, resolve across profiles
- **MCP stdio config blocking** — suspicious configurations blocked before probe execution
- **Secret redaction in API debug dumps** — prevents credential leakage in logs

## Upgrade Instructions

```bash
cd ~/compiled/autolycus-agent
git pull origin main --tags
./venv/bin/pip install -e .
```

**Configuration migration:** If you have custom settings referencing "hermes" in their names, update them to use "lycus". The system will accept both during this transition period, but the old names may be removed in a future release.

**Cron jobs:** Existing cron jobs are automatically compatible with the new `on_success` field (defaults to empty list). To enable state-driven chaining, add the field via CLI:
```bash
lycus cron update <job_id> --on-success <downstream_job_id>
```

If you prefer Hermes, use Hermes. If you want what Autolycus offers — state-driven job chaining, self-pruning memory, unified compression, HIAL personality modes, and the architectural decisions that make them possible — then this release is for you.
